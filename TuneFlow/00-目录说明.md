# TuneFlow 目录说明

当前推荐直接按步骤使用：

- `01-打开歌单导出GUI.bat`
  第 1 步，从 myhkw 拉取歌单并导出 `songs.json`。
- `02-打开歌曲视频复核GUI.bat`
  第 2 步，自动扫描候选结果后，手动补链接并导出 `songs.matched.json`。
- `02-一键自动匹配视频.bat`
  第 2 步的自动版，直接批量生成 `songs.matched.json`，之后再人工复核。
- `03-一键生成播放器映射.bat`
  第 3 步，把 `songs.matched.json` 生成成 `songs.player-map.json` 和 `songs.player-map.js`。
- `tuneflow.html`
  最终播放器页面，默认从云端读取 `songs.player-map.js`。

核心中间产物：

- `songs.json`
  第 1 步导出的歌曲列表。
- `songs.matched.json`
  第 2 步补完视频后的匹配结果。
- `songs.player-map.json`
  第 3 步生成的播放器映射 JSON。
- `songs.player-map.js`
  第 3 步生成的浏览器可直接加载版本。

核心源码：

- `myhkw-song-exporter.html`
- `song-video-matcher.html`
- `export-myhkw-songs.mjs`
- `auto-match-bilibili.mjs`
- `export-player-map.mjs`

兼容入口：

- `打开导出GUI.bat`
- `一键导出songs-json.bat`
- `打开歌曲视频匹配GUI.bat`
- `一键自动匹配视频.bat`
- `一键生成播放器映射.bat`

历史版本已归档到：

- `archive/versions/`

补充说明：

- 公开提交到 GitHub 前，不要把你自己的 `myhkw key` 写死在脚本里。
- 已生成的 `songs.json`、`songs.matched.json`、`songs.player-map.*` 也可能带有历史私有数据，提交前请先检查或重新生成。
